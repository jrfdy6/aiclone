## First-pass draft

Most teams treat persona files like a mood board. We treat them like code. Nothing reaches canon until it survives the same operating system that runs Brain, Ops, weekly plans, and drafts.

The path is six deliberate steps:
1. **Save the thought** — capture the quote, belief, or lived example exactly as it happened, artifact link included.
2. **Queue for promotion** — drop it into `persona_deltas` with the proof attached so Workspace and Brain can see the same backlog.
3. **Reroute** — Workspace approvals count, but anything unresolved moves into Brain’s ranked review queue with context on tension, agreement, or missing evidence.
4. **Commit to canon** — only after approval do we write into `knowledge/persona/feeze/**`, and even then the entry cites the artifact so future audits can prove it.
5. **Bundle sync** — local bundles update so the next retrieval pass reads the change immediately instead of waiting for a deploy.
6. **Bundle-first generation** — drafts, planners, and automations pull from those synced bundles, so nothing new hits the feed without the approved delta behind it.

We built that path because we were tired of “good quotes” drifting into drafts that nobody could prove. During last week’s review we rejected three snippets that felt inspiring but had no receipts. Under the old system they would have gone straight into copy and rewritten history. Under this one they died in queue, and the operators who needed the truth still trust the file they’re reading.

If you want AI work to feel accountable, stop treating persona memory like a notion doc. Treat it like an operating system with gates, logs, and rollbacks. Otherwise you’re just improvising brand every time you publish.
