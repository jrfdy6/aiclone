# Source Intelligence Contract

This document defines how external learning material should enter the AI Clone brain.

Examples:
- videos
- screenshots
- transcripts
- notes from other operator systems
- reference workflows

These sources can shape future workspace design.

They should not bypass digestion and go straight into PM or identity packs.

## 1. Purpose

Source intelligence exists to turn outside material into useful internal signal.

It should help the system answer:
- what is relevant
- which workspace it affects
- whether it is actionable now
- whether it is strategic only
- whether it should change a workspace pack later

## 2. Folder Structure

Source intelligence should live under:

- [knowledge/source-intelligence/README.md](/Users/neo/.openclaw/workspace/knowledge/source-intelligence/README.md)

Recommended substructure:
- `raw/`

## 3. Processing Stages

### Raw

Raw source material:
- transcript
- screenshot
- notes
- link

Raw files are not canonical memory.

### Digest

A digest is the first structured interpretation.

It should answer:
- what this source says
- what workspaces it might affect
- what signal types are present
- what should be ignored for now

### Promotion

Promotion is the deliberate step where a digest changes:
- Chronicle
- durable memory
- standup agenda
- PM board
- workspace identity packs

## 4. Routing Rules

### Route to Chronicle

Use when:
- the source changes current thinking
- the source should influence the next standup
- the insight may be lost before the next memory pass

### Route to Weekly Review / Saturday Vision

Use when:
- the source is strategic
- the source shapes long-term direction
- the source should be weighed, not acted on immediately

### Route to workspace standup

Use when:
- the source is clearly relevant to one workspace
- the source might change execution or local operating rules

### Route to PM board

Only when:
- the source implies a concrete next action
- ownership is clear
- the step is actually executable now

### Route to identity packs

Only when:
- the insight is durable
- the owner intent is clear
- the source is not just hype or inspiration
- standup review confirms the promotion

## 5. Role Mapping

### Neo
- sees whether the source matters system-wide
- decides whether it should change portfolio priorities

### Jean-Claude
- decides whether the source should change execution or SOP quality
- translates strategic insight into operational form

### Yoda
- interprets long-horizon strategic meaning
- is especially important for `FEEZIE OS`, identity, brand, and direction

### Workspace agents
- should not consume raw source intelligence directly by default
- should consume digested, workspace-scoped interpretation

## 6. Executive Triage Rhythm

When a source is important enough to shape direction, the executive conversation should include:
- `Yoda`
- `Neo`
- `Jean-Claude`

Their jobs are different:
- `Yoda` asks what the source means for long-horizon direction
- `Neo` asks what it changes across the whole system or portfolio
- `Jean-Claude` asks what should happen operationally because of it

This triage should usually happen in:
- executive standup
- weekly review
- Saturday vision sync when the signal is strategy-only

## 7. Multi-Route Rule

One source may validly route to more than one place.

For example:
- persona canon
- canonical memory
- FEEZIE OS standup
- workspace standup
- PM board

That is allowed when the routes are justified by the same executive interpretation.

What should not happen is raw source material writing itself into all lanes without that executive filter.

## 8. FEEZIE OS Rule

`FEEZIE OS` should receive more source intelligence than the other workspaces because:
- it is identity-adjacent
- it is public-facing
- it is still evolving

But even there:
- raw source material should not directly rewrite the lane
- Yoda should pressure-test strategic meaning first
- Jean-Claude should turn it into operational structure second

## 9. What Good Looks Like

A good source-intelligence pipeline does this:

1. saves the raw source
2. creates a digest
3. appends a high-signal Chronicle entry if needed
4. feeds the right standup
5. promotes only the durable parts
6. leaves the rest as advisory reference

## 10. What Bad Looks Like

Bad source-intelligence behavior:
- one video instantly rewrites the PM board
- one screenshot rewrites a workspace soul
- raw inspiration turns into fake urgency
- multiple workspaces inherit the same generic advice
- the system starts copying other systems instead of adapting them

## 11. Recommended Next Step

Use source intelligence to shape:
- weekly review
- Saturday vision sync
- FEEZIE OS evolution
- future workspace-pack refinement

Do not use it as a direct replacement for:
- PM board truth
- local execution evidence
- workspace-specific real-world feedback
